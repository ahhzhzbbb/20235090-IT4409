Introduction

This page is a compact guide to core frontend concepts for beginners.

You can read each section in order, or jump directly from the left navigation.

<!DOCTYPE html>
HTML Basics

HTML gives structure to a page using semantic elements like headings, sections, and links.

Good structure helps both users and search engines understand your content.

  • Use one main heading to define the page topic.
  • Use lists when you have grouped items.
<h1>Welcome</h1>
CSS Selectors

CSS selectors target elements so you can style them consistently.

Start with simple selectors, then move to class combinations as your UI grows.

  • Element selector: targets all tags of one type.
  • Class selector: targets reusable visual patterns.
.card { border-radius: 8px; }
Flexbox Layout

Flexbox is great for one-dimensional layouts such as rows, columns, and navigation bars.

It simplifies spacing, alignment, and responsive rearrangement with minimal code.

  • Use display: flex on the parent element.
  • Control alignment with justify-content and align-items.
.row { display: flex; gap: 12px; }
Responsive Design

Responsive design makes interfaces usable on phones, tablets, and desktops.

Combine fluid units, flexible images, and media queries to adapt layout size.

  • Prefer percentages and rem for scalable spacing.
  • Test breakpoints where content starts to feel cramped.
@media (max-width: 768px) { ... }
Accessibility Tips

Accessible pages are easier for everyone to use, including people with disabilities.

Small improvements such as better contrast and labels have a big impact.

  • Use descriptive link text instead of “click here”.
  • Always provide alt text for meaningful images.
<img src="avatar.jpg" alt="Profile photo">
